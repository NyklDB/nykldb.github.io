Class: NyckelDB

NyckelDB(appID, tableTitle)

Create a new instance of NyckelDB

Constructor

new NyckelDB(appID, tableTitle)

Parameters:
Name Type Description
appID string

the name of, or unique identifier of the app, especially important to differentiate multiple apps hosted on the same website domain or being tested on the same localhost server, to avoid in-browser localstorage conflicts. NyckelDB tables with the same appID can "see" each other, i.e. you can link them
tableTitle string

the name of the new table. This is what the table will be referred to inside your app. You cannot have 2 tables with the same appID and same tableTitle, as they will overwrite each other
